# Security Policy

## Scope

This repository is a Codex plugin bundle and supporting documentation. It is not a hosted service, but security issues may still matter if they could lead to unsafe code generation guidance, credential disclosure, malicious plugin behavior, or risky installation instructions.

## Supported Versions

Security fixes are only guaranteed for the latest state of the default branch.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Older commits/releases | No |

## Reporting A Vulnerability

Please do not open a public GitHub issue for suspected security vulnerabilities.

Instead, report privately by emailing:

- `tim@millrace.ai`

If GitHub private vulnerability reporting is enabled for this repository, that is also an acceptable channel.

Please include:

- a clear description of the issue
- affected files or paths
- reproduction steps or a proof of concept
- impact assessment
- any suggested mitigation, if you have one

## Response Expectations

- Initial acknowledgment target: within 5 business days
- Status update target: within 10 business days when the report is valid and actionable

## Disclosure

Please allow reasonable time for validation and remediation before public disclosure.
